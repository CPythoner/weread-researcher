# 单书分析命令

## 模块目标

围绕一本书完成从总结、深挖、行动转化到提问复盘的完整分析。

## 命令列表

```text
/book-summary 《书名》
/book-deepdive 《书名》
/book-to-action 《书名》
/book-questions 《书名》
```

## `/book-summary`

### 目标

整理一本书的划线和笔记，生成结构化阅读笔记。

### 输出

```markdown
# 《书名》阅读总结

## 1. 一句话总结
## 2. 这本书解决什么问题
## 3. 核心观点
## 4. 我的高频关注点
## 5. 代表性划线
## 6. 我的想法
## 7. 可复用金句
## 8. 行动清单
```

## `/book-deepdive`

### 目标

分析这本书对用户的意义，而不是只总结内容。

### 输出

```markdown
# 《书名》深度分析

## 1. 作者真正想回答的问题
## 2. 作者的核心假设
## 3. 用户为什么会被这本书击中
## 4. 最值得保留的观点
## 5. 最值得质疑的观点
## 6. 与已读书籍的关联
## 7. 对当前项目 / 人生阶段的启发
```

## `/book-to-action`

### 目标

把一本书转成具体行动。

### 输出

```markdown
# 《书名》行动转化

## 1. 可执行原则
## 2. 7 天行动清单
## 3. 可能遇到的阻力
## 4. 验证指标
## 5. 复盘问题
## 6. 适合写入每日计划的任务
```

## `/book-questions`

### 目标

根据书籍内容和用户笔记，生成深度追问。

### 输出

```markdown
# 《书名》深度问题

## 1. 理解问题
## 2. 反思问题
## 3. 行动问题
## 4. 质疑问题
## 5. 延伸问题
## 6. 给未来自己的问题
```

## 质量要求

- 必须区分“作者观点”和“用户自己的想法”。
- 代表性划线要服务于分析，不要堆砌摘录。
- 行动建议要具体到可以执行。
- 深度问题要避免泛泛而谈。
