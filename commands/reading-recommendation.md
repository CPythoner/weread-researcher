# 阅读推荐命令

## 模块目标

基于用户真实阅读轨迹、划线密度、笔记主题、认知缺口和当前问题，推荐下一本真正该读的书。

推荐不是“猜你喜欢”，而是帮助用户完成阅读闭环：

```text
已有阅读 → 当前问题 → 认知缺口 → 推荐书籍 → 阅读计划 → 输出任务
```

## 命令列表

```text
/next-book
/recommend-by-goal 目标
/recommend-by-gap
/recommend-for-question 当前问题
/recommend-opposite
/recommend-topic 主题
/recommend-for-content
/reading-roadmap 主题或目标
/shelf-cleanup
```

## `/next-book`

### 目标

根据最近阅读、在读书籍、划线密度和笔记主题，推荐下一本最适合读的书。

### 输出

```markdown
# 下一本书推荐

## 1. 当前阅读状态判断
## 2. 最推荐的一本
## 3. 推荐理由
## 4. 它承接了你之前哪本书 / 哪个主题
## 5. 适合怎么读：精读 / 泛读 / 跳读 / 主题阅读
## 6. 读完后做什么
```

## `/recommend-by-goal`

### 目标

用户输入一个目标，根据已有阅读基础推荐分阶段书单。

### 示例

```text
/recommend-by-goal 我想系统学习 AI Agent
/recommend-by-goal 我想做个人品牌
/recommend-by-goal 我想提高决策能力
```

### 输出

```markdown
# 按目标推荐：{{goal}}

## 1. 目标拆解
## 2. 当前基础判断
## 3. 入门书
## 4. 进阶书
## 5. 深挖书
## 6. 反方书
## 7. 行动书
## 8. 阅读顺序
```

## `/recommend-by-gap`

### 目标

先分析知识结构缺口，再推荐补缺口的书。

### 输出

```markdown
# 按认知缺口推荐

## 1. 当前知识结构
## 2. 主题缺口
## 3. 难度缺口
## 4. 视角缺口
## 5. 实践缺口
## 6. 反方缺口
## 7. 每个缺口对应推荐书
```

## `/recommend-for-question`

### 目标

根据用户当前困惑推荐书，而不是只按主题推荐。

### 示例

```text
/recommend-for-question 我现在很焦虑，不知道下一步做什么
/recommend-for-question 我想从技术转向独立开发
/recommend-for-question 我想提升表达能力
```

### 输出

```markdown
# 按当前问题推荐

## 1. 当前问题拆解
## 2. 你真正需要补的不是哪类知识
## 3. 推荐书单
## 4. 每本书解决什么问题
## 5. 阅读顺序
## 6. 读后任务
```

## `/recommend-opposite`

### 目标

推荐不一定舒服，但能打破当前认知舒适区的书。

### 输出

```markdown
# 反向推荐

## 1. 你的阅读舒适区
## 2. 这个舒适区可能带来的局限
## 3. 反向挑战书单
## 4. 每本书会挑战你什么
## 5. 阅读提醒
```

## `/recommend-topic`

### 目标

围绕一个主题生成进阶书单。

### 输出

```markdown
# 主题进阶推荐：{{topic}}

## 0 层：概念入门
## 1 层：框架建立
## 2 层：经典理论
## 3 层：案例实践
## 4 层：反方观点
## 5 层：输出任务
```

## `/recommend-for-content`

### 目标

专门为自媒体内容创作推荐书。

### 输出

```markdown
# 内容创作者书单推荐

## 1. 适合拆选题的书
## 2. 适合做金句卡的书
## 3. 适合做深度文章的书
## 4. 适合做短视频的书
## 5. 适合做系列内容的书
## 6. 每本书可拆出的内容方向
```

## `/reading-roadmap`

### 目标

输出一个长期阅读路线，而不是几本书。

### 输出

```markdown
# 阅读路线图：{{topic_or_goal}}

## 第 1 阶段：建立基础认知
## 第 2 阶段：补齐关键模型
## 第 3 阶段：阅读经典文本
## 第 4 阶段：结合项目实践
## 第 5 阶段：形成自己的方法论

每个阶段包含：
- 推荐书籍
- 阅读目标
- 输出任务
- 判断标准
```

## `/shelf-cleanup`

### 目标

帮用户判断书架里的书哪些该继续读，哪些该暂时放下。

### 输出

```markdown
# 书架清理建议

## 1. 立刻读
## 2. 稍后读
## 3. 跳读
## 4. 暂时放下
## 5. 可以删除 / 忽略
## 6. 书架结构优化建议
```

## 推荐质量要求

- 不能只给热门书名，必须说明“为什么现在推荐”。
- 必须关联用户已有阅读证据。
- 推荐结果要包含阅读方法和读后任务。
- 需要至少有一本“反向挑战书”。
- 需要区分“喜欢读”和“应该读”。
