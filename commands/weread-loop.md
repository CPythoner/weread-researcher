# `/weread-loop` 总控工作流

## 目标

执行完整阅读闭环：

```text
读过什么 → 总结什么 → 多视角洞察 → 看见自己 → 发现缺口 → 推荐下一本 → 继续阅读 → 导出 HTML
```

## 参数

```text
/weread-loop [范围或主题] [--lens default|deep|decision|content] [--html] [--markdown] [--obsidian] [--xhs]
```

示例：

```text
/weread-loop
/weread-loop 最近一周
/weread-loop 最近三个月 --lens deep --html
/weread-loop AI Agent --lens decision
/weread-loop 独立开发 --lens content --html
```

## 默认设置

| 参数 | 默认值 |
|---|---|
| 时间范围 | 最近 30 天 |
| 洞察组合 | default |
| 输出格式 | Markdown + HTML 摘要 |
| 推荐数量 | 5 本 |
| 继续阅读计划 | 7 天 |

## 执行步骤

### Step 0. Collect Reading Data

读取并标准化微信读书数据：

- 书架
- 已读
- 在读
- 想读
- 阅读时长
- 阅读进度
- 划线
- 笔记 / 想法
- 书评
- 阅读时间线

输出结构：

```json
{
  "range": "最近30天",
  "books": [],
  "highlights": [],
  "notes": [],
  "reviews": [],
  "timeline": [],
  "topics": []
}
```

### Step 1. 读过什么

输出：

- 阅读概览
- 书籍状态
- 阅读投入排行
- 笔记密度排行
- 高频主题
- 当前阅读状态判断

### Step 2. 总结什么

输出：

- 最近阅读的核心观点
- 最值得保留的划线
- 用户自己的关键想法
- 可行动线索
- 可输出线索

### Step 3. 多视角洞察

根据 `--lens` 执行洞察组合。

默认组合：

```text
global-scan
meaning-radar
core-conflict
value-clarification
blindspot-explore
third-person-view
personal-flywheel
contrarian-insights
```

### Step 4. 看见自己

将所有洞察压缩为自我画像：

- 当前的你
- 真正关心的问题
- 核心追求
- 当前主要矛盾
- 隐藏焦虑
- 逃避与渴望
- 下一阶段最应该面对的问题

### Step 5. 发现缺口

输出：

- 主题缺口
- 难度缺口
- 视角缺口
- 行动缺口
- 输出缺口
- 决策缺口

### Step 6. 推荐下一本

推荐 5 类书：

| 类型 | 作用 |
|---|---|
| 最该读 | 当前阶段最值得读 |
| 补缺口 | 弥补认知短板 |
| 反向挑战 | 打破舒适区 |
| 内容输出 | 适合文章 / 视频 / 卡片 |
| 轻松过渡 | 保持阅读节奏 |

每本书需要包含：

- 推荐理由
- 与已有阅读的关联
- 适合怎么读
- 读后任务

### Step 7. 继续阅读

生成 7 天计划：

- 每天读什么
- 带着什么问题读
- 重点章节
- 笔记任务
- 输出任务
- 下次复盘时间

### Step 8. 导出 HTML

使用 `templates/reading-loop-report.html` 的设计风格导出报告。

## 输出大纲

```markdown
# 阅读闭环报告

## 0. 一句话洞察
## 1. 读过什么
## 2. 总结什么
## 3. 多视角洞察
## 4. 看见自己
## 5. 发现缺口
## 6. 推荐下一本
## 7. 继续阅读
## 8. HTML 导出
```

## 洞察证据要求

每个洞察必须尽量包含证据：

```markdown
### 当前张力：长期主义 vs 即时反馈

**证据**：
- 近期多次划线涉及“长期积累”“复利”“持续输出”。
- 但笔记中也反复出现“数据反馈”“短期效果”“不确定是否值得继续”。

**解释**：
你并不是不相信长期，而是缺少稳定的短期反馈机制。

**下一步**：
为长期项目设计一个每周可见的小指标。
```

## 注意事项

- 不要把推断写成绝对结论。
- 不要进行心理诊断。
- 不要只总结书，要区分书的观点和用户自己的想法。
- 不要只推荐热门书，要解释为什么现在推荐。
