# 多视角洞察命令

## 模块目标

从不同镜头理解同一批阅读记录。这个模块参考 flomo 式洞察体验，但命名、结构和输出都做成 `weread-mirror` 自己的镜头系统。

## 复盘整理类

### `/growth-evidence`

从笔记中找出值得肯定的小进步。

输出：

```markdown
# 成长证据

## 1. 你已经做对了什么
## 2. 最近出现的进步信号
## 3. 哪些笔记说明你已经理解更深
## 4. 值得继续强化的行为
## 5. 今天可以肯定自己的一句话
```

### `/action-guide`

把困惑和线索转成具体行动。

输出：

```markdown
# 行动指南

## 1. 当前最重要的问题
## 2. 可以立刻行动的 3 件事
## 3. 暂时不需要做的事
## 4. 阻力与应对
## 5. 7 天行动计划
```

### `/deep-questions`

根据笔记生成尖锐但有价值的问题。

输出：

```markdown
# 深问自己

## 1. 你一直绕开的 3 个问题
## 2. 最值得现在面对的问题
## 3. 如果诚实回答，答案可能是什么
## 4. 可以写进日记的追问
```

### `/theme-extension`

从已有主题延展出新的研究方向。

输出：

```markdown
# 主题延伸

## 1. 当前已形成的主题
## 2. 每个主题可以继续追问什么
## 3. 可以发展成文章 / 项目 / 课程的方向
## 4. 下一步研究建议
```

### `/inner-assets`

找出用户已经拥有的能力、经验和支点。

输出：

```markdown
# 内在资源

## 1. 你已经拥有的能力
## 2. 你已经积累的知识资产
## 3. 你可以依靠的经验
## 4. 可以组合成作品的资源
```

### `/contrarian-insights`

找出反直觉但有价值的观点。

输出：

```markdown
# 反常识洞见

## 洞见 1
- 标题
- 解释
- 阅读证据
- 适合输出的平台

## 洞见 2
...
```

### `/global-scan`

对全部笔记做综合扫描。

输出：

```markdown
# 全局扫描

## 1. 长期主线
## 2. 近期转向
## 3. 隐藏主题
## 4. 当前阶段判断
## 5. 一句话洞察
```

## 自我觉察类

### `/meaning-radar`

分析笔记中的意义来源和动力核心。

### `/influence-map`

找出反复影响用户的作者、人物、理论和榜样。

### `/bottom-question`

找出用户心底反复追问的问题。

### `/acceptance-action`

用接纳与行动视角区分：哪些事该接纳，哪些事该行动。

### `/blindspot-explore`

找出用户看不见但可能重要的真相。

### `/third-person-view`

像朋友一样观察用户当前状态。

### `/life-script`

像导演看剧本一样分析用户正在书写的人生叙事。

### `/thought-trap-check`

识别笔记中的常见思维陷阱，如非黑即白、灾难化、过度概括、读心术、应该化等。

### `/personality-hints`

从笔记中分析人格倾向线索，仅作为趣味参考，不作为定论。

统一输出：

```markdown
# {{lens_name}}

## 1. 一句话洞察
## 2. 阅读证据
## 3. 解释
## 4. 可能盲区
## 5. 下一步行动
```

## 思维决策类

### `/personal-flywheel`

构建个人复利飞轮：需求、优势、资产、正反馈、阻塞点、启动动作。

### `/core-conflict`

识别当前最核心冲突。

### `/value-clarification`

澄清真正重要的价值。

### `/inversion-thinking`

用逆向思考分析失败路径和应该避免的错误。

### `/second-order-thinking`

分析一阶后果、二阶后果和长期影响。

## 大师视角类

### `/munger-view`

用查理·芒格式多元思维模型分析阅读和决策问题。

### `/aristotle-view`

用第一性原理、目的论和德性视角分析用户长期追求。

### `/seneca-view`

用斯多葛控制二分法分析焦虑、行动和内耗。

### `/eurich-view`

用塔莎·尤里奇的自我觉察视角，分析内部视角、外部视角和成长盲区。

## 质量要求

- 每个视角都必须有阅读证据。
- 不要把“洞察”写成鸡汤。
- 不要做心理诊断。
- 要给出具体下一步行动。
- 大师视角不能只是模仿名人语气，要使用其核心方法论。
