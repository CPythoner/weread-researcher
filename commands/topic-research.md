# 主题研究命令

## 模块目标

跨多本书和多条笔记，围绕一个主题生成知识地图、观点对比、缺口分析和学习路径。

## 命令列表

```text
/topic-map 主题
/topic-compare 主题
/reading-gap 主题
/learning-path 主题
```

## `/topic-map`

### 目标

从用户全部书籍、划线和笔记中，围绕一个主题生成知识地图。

### 输出

```markdown
# 主题地图：{{topic}}

## 1. 相关书籍
## 2. 相关划线和笔记
## 3. 核心概念
## 4. 观点分组
## 5. 主题结构图
## 6. 可继续研究的问题
```

## `/topic-compare`

### 目标

对比多本书对同一主题的不同看法。

### 输出

```markdown
# 主题观点对比：{{topic}}

## 1. 多本书的共同结论
## 2. 不同作者的分歧
## 3. 观点成立的边界
## 4. 用户当前更倾向哪一方
## 5. 综合判断
## 6. 还需要补充的反方观点
```

## `/reading-gap`

### 目标

分析用户在某个主题下的知识结构缺口。

### 输出

```markdown
# 阅读缺口分析：{{topic}}

## 1. 已有基础
## 2. 主题缺口
## 3. 难度缺口
## 4. 视角缺口
## 5. 行动缺口
## 6. 反方缺口
## 7. 推荐补充路径
```

## `/learning-path`

### 目标

为一个主题生成系统学习路线。

### 输出

```markdown
# 学习路径：{{topic}}

## 1. 当前基础判断
## 2. 入门阶段
## 3. 进阶阶段
## 4. 深入阶段
## 5. 实战阶段
## 6. 输出阶段
## 7. 30 天学习计划
```

## 质量要求

- 主题研究必须跨书，而不是只总结一本书。
- 必须说明哪些内容来自用户已有阅读，哪些是建议补充。
- 对比观点时要写清楚适用边界。
- 学习路径要结合用户当前基础，而不是通用书单。
