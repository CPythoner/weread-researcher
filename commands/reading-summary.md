# 阅读汇总命令

## 模块目标

围绕时间维度生成阅读日报、周报、月报和年度复盘，回答：这段时间读了什么、留下了什么、发生了什么变化、下一步该怎么读。

## 命令列表

```text
/reading-daily
/reading-weekly
/reading-monthly
/reading-yearly
```

## `/reading-daily`

### 适用场景

每天结束或第二天早晨快速复盘。

### 输入

- 当天阅读记录
- 当天新增划线
- 当天新增笔记 / 想法
- 当天阅读进度变化

### 输出

```markdown
# 今日阅读简报

## 1. 今天读了什么
## 2. 今天最重要的 3 条划线
## 3. 今天自己的想法
## 4. 今天获得的一个启发
## 5. 可以立刻做的一件小事
## 6. 明天继续读什么
```

## `/reading-weekly`

### 适用场景

每周复盘阅读投入、主题变化和行动线索。

### 输出

```markdown
# 本周阅读周报

## 1. 本周阅读概览
- 已读 / 在读 / 想读
- 阅读时长
- 笔记数量
- 划线数量

## 2. 本周高频主题
## 3. 本周最重要的 5 条笔记
## 4. 本周认知变化
## 5. 本周行动线索
## 6. 下周阅读建议
```

## `/reading-monthly`

### 适用场景

月度主题复盘和下一阶段阅读规划。

### 输出

```markdown
# 月度阅读报告

## 1. 月度阅读统计
## 2. 月度代表书籍
## 3. 月度主题地图
## 4. 月度认知变化
## 5. 月度内容资产
## 6. 月度阅读缺口
## 7. 下月阅读策略
```

## `/reading-yearly`

### 适用场景

年度复盘、公开分享、个人知识资产沉淀。

### 输出

```markdown
# 年度阅读复盘

## 1. 年度阅读概览
## 2. 年度关键词
## 3. 年度代表书籍
## 4. 年度心路变化
## 5. 年度知识结构
## 6. 年度行动转化
## 7. 年度内容资产
## 8. 下一年阅读方向
```

## 质量要求

- 不只统计数量，要解释“为什么这些书重要”。
- 区分“读过很多”和“真正读进去”。
- 优先引用用户自己的笔记，而不是只复述书籍内容。
- 每次输出都给出下一步阅读建议。
